# Data Builder 구역 제약 조건

## 1. Data Leakage (데이터 누수) 원천 차단
* 타겟 변수인 '성공 여부(KPI)' 또는 미래 시점의 실적 데이터가 [속성], [트렌드/IP] 노드의 초기 피처(Feature)로 절대 스며들지 않도록 구조를 설계하십시오.
* KPI 정보는 오직 [상품] 노드의 초기화 및 타겟 레이블(Label)로만 분리되어 사용되어야 합니다.

## 2. 그래프 구조화 엄격성
* 이기종 그래프(Heterogeneous Graph) 구축 시 노드 타입은 반드시 4가지(상품, 속성, 트렌드, 영수증)로만 제한합니다.
* 임의의 노드 타입이나 무분별한 엣지 생성을 금지하여 연산 폭발을 방지하십시오.
